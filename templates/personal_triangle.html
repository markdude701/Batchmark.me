Project Scope

 

Project Description

           

            Each week, Triangle Fraternity has a weekly meeting, and during this meeting, a weekly agenda is shown to the brother, which is comprised of the reports of the eboard and chair positions, New Business, Old Business, Misc. Business, and Important Dates. Each week, prior to the meeting, the Scribe chair must review the brothers’ submissions, and then import those submissions into an agenda.

In the past, Triangle Fraternity has had issues with the efficiency of the Scribe chair, and the manner in which they create the agenda. Even though the reports are only submitted by the eboard, it is difficult to collect reports and create the agenda prior to a meeting. This can be due to the time the reports are due, or the lack of report submissions in general. The current Scribe chair can choose the best time to have brother reports due each week, but for some brothers, the select time is inconvenient.

            Asking members of the e-board, those who would be directly using this function, what they would like to see in an improved report submission system, resulted in these responses:

 

“... automatically emails out before the meeting”, Tyler Traeger, President

“It would be really nice if I didn’t have to turn in reports so early”, Nathan Phillips, Brother

“Saturday seems pretty early”, Drew Sorensen, President Elect (Brother)

 

            Having a system in place that can automatically  generate a report when the Scribe is ready to create the agenda  would eliminate the “too early” problem. Giving the brother the ability to submit a report several minutes before the meeting, and only relying on the current Scribe chair to update the report a few minutes prior to the meeting would be efficient. The Scribe chair would review the document, prior to the meeting, and would change anything if needed.

 

User Stories

 

As a scribe at Triangle Fraternity, it’s important that I save time and inform our fellow brothers through automation of our brother reports system by use of a database, google forms, and the FormPublisher add on.

 

As a brother of the Triangle Fraternity, I would like a response when my reports get received by the scribe so that my voice gets heard and our agenda during meetings runs smoothly.

Suggested Solution / Software Packages:

            In the order to have an automatic process, a software solution must be found. Currently Triangle Fraternity, The Behrend Chapter, uses a Google Form to submit reports. The report information is then collected into a Google Sheet. After the reports are submitted, the Scribe chair must manually update a master template document.

            Using the abilities of a Google Form addon called FormPublisher, alongside a Javascript that is ran in Google Forms, an agenda can be created based on a template, as well as email responses for submissions. FormPublisher is an addon for the Google Apps Suite (Docs, Sheets, Drive, etc), that creates response and text documents based on the user’s input on the Google Form.

            The user’s input is collected into a Google sheet, and when the user successfully submits a report, they are emailed a confirmation email with their submission. Each document sent to the brothers is created by FormPublisher, using a standard submission template.

            Once the responses are collected, the Scribe chair would run a script in Google Drive in order to generate a master agenda document prior to the meeting, based off a template document. There currently is no way to achieve this with an application, using the Google architecture.

            The Javascript, ran in Google Drive, will output a final agenda document in an “Output” directory. The agenda will be created from the information submitted by the user, such as name and chair, and will follow a template document, that uses %VAR% as variables.

Data Model:

            The first source of input from the user is in the Postion_Info table. The user input their information, which determines the position within the agenda the brother’s report will reside.  The position of the brother will determine the order within the agenda.

The user then inputs their agenda content. The user’s report, New_Bus, Old_Bus, Misc_Bus, and Dates create the content of the agenda. The agenda must be reviewed by the Scribe prior to publishing. The brother must also submit a report, if they want to have a submission in the agenda.

function appendToDoc(src, dst) {
  for (var i = 0; i < src.getNumChildren(); i++) {
    appendElementToDoc(dst, src.getChild(i));
  }
